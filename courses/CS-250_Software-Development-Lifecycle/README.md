# CS-250: Software Development Lifecycle

## Course Description

This course provides comprehensive coverage of the Software Development Lifecycle (SDLC) with a primary focus on Agile methodologies and modern software development practices. Students learn to translate business requirements into technical user stories, manage development workflows, and work effectively within cross-functional teams.

## Key Topics

- **Agile Methodologies**: Scrum framework, sprints, and iterative development
- **SDLC Phases**: Requirements gathering, design, implementation, testing, deployment, and maintenance
- **User Stories**: Writing effective user stories, acceptance criteria, and story mapping
- **Team Collaboration**: Team charters, roles and responsibilities, communication protocols
- **Project Management**: Sprint planning, backlog management, and progress tracking
- **Requirements Analysis**: Translating client needs into actionable development tasks

## Tools & Technologies

- **Documentation**: Microsoft Word for team charters and requirements documentation
- **Spreadsheets**: Microsoft Excel for user story management and backlog tracking
- **Project Management**: Gantt charts for timeline visualization and project planning
- **Collaboration Tools**: Team charter frameworks and communication templates

## Major Assignments

### 1. Agile Team Charter
**Files**: `CS 250 Agile Team Charter Heath Davis.docx`

Development of a comprehensive team charter establishing team structure, roles, responsibilities, and collaboration protocols for an Agile development team.

### 2. User Stories Assignment
**Files**: `CS 250 Module Three User Stories Assignment Heath Davis.xlsx`

Creation of detailed user stories with acceptance criteria, story points, and priority rankings based on client requirements gathered from stakeholder meetings.

### 3. Client Requirements Analysis
**Files**:
- `CS 250 Initial Client Meeting Animation Text-Only Version.docx`
- `CS 250 Product Owner and User Focus Group Animation Text-Only Version.docx`

Analysis of client meeting transcripts to extract functional and non-functional requirements for software development projects.

## Key Concepts Demonstrated

- Translation of business requirements into technical specifications
- Agile sprint planning and backlog prioritization
- Cross-functional team organization and communication
- Stakeholder engagement and requirements elicitation
- User-centered design thinking
- Iterative development processes

## Skills Acquired

- Writing clear, testable user stories with acceptance criteria
- Creating and managing product backlogs

- Conducting requirements analysis from client interactions
- Applying Agile principles to real-world development scenarios
- Estimating and prioritizing development work
