UML Class Diagram: DriverPass System

Object-Oriented Design - CS 255 System Analysis and Design

Class Diagram Description

This class diagram represents the object-oriented structure of the DriverPass system. The diagram shows the main classes, their attributes, and the relationships between them. The User class serves as the parent class with five specialized subclasses: Customer, Secretary, Driver, ITOfficer, and Owner, demonstrating inheritance relationships shown by the triangular arrow heads.

Key relationships include Customer having one Package, Customer making multiple Appointments, Customer taking multiple TestResults, and Customer making multiple Payments. Driver is assigned to multiple Appointments and creates multiple LessonNotes. Vehicle is used in multiple Appointments. OnlineTest contains multiple TestQuestions and generates multiple TestResults. DMVUpdate updates multiple TestQuestions to maintain current compliance with DMV requirements.

The ActivityLog class tracks all user actions in the system for accountability and security purposes. Each class includes relevant attributes that support the functional requirements identified in the DriverPass interview, including user management, appointment scheduling, online testing, payment processing, and DMV integration capabilities.

Relationship Notation

  • Inheritance: Hollow triangle arrow indicates subclass inheriting from parent class
  • Association: Solid line with filled arrow shows directional relationship between classes
  • Multiplicity: 1 indicates one instance, * indicates many instances
  • Attributes: Listed with minus sign indicating private visibility
