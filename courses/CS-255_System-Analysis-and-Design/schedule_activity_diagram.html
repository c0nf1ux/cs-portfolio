UML Activity Diagram: Schedule Appointment Use Case

DriverPass System - CS 255 System Analysis and Design

Activity Diagram Description

This activity diagram illustrates the complete workflow for scheduling a driving lesson appointment in the DriverPass system. The process includes user authentication, retrieving and displaying available appointment options, validating selections for scheduling conflicts, processing payment, creating the appointment record, and sending confirmations to both the customer and assigned driver. Error handling paths ensure users receive appropriate feedback when authentication fails, scheduling conflicts exist, or payment processing encounters issues.
