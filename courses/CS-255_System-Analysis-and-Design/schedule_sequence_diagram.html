UML Sequence Diagram: Schedule Appointment Use Case

DriverPass System - CS 255 System Analysis and Design

Sequence Diagram Description

This sequence diagram illustrates the temporal interactions between system components during the appointment scheduling process. The diagram shows five participants: Customer, System Interface, Application Server, Database, and Payment Gateway. The sequence begins with the customer requesting to schedule an appointment. The system authenticates the user by querying the database for credentials. Upon successful authentication, the system retrieves available appointment times by checking driver, vehicle, and existing appointment availability in the database. Available options are displayed to the customer who selects their preferred date, time, driver, and vehicle. The system validates the selection by checking for scheduling conflicts in the database. After validation, the customer enters payment information which is processed through an external payment gateway. Upon payment confirmation, the application server creates the appointment record in the database, logs the activity, and generates confirmation details. The system displays the confirmation to the customer and sends email notifications to both the customer and the assigned driver. The vertical dashed lines represent lifelines showing each participant's existence over time, while activation boxes show when each participant is actively processing. Solid arrows represent synchronous messages, and dashed arrows represent return messages or responses.
